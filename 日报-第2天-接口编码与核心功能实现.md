## 今日完成工作

### 1. 项目框架搭建

基于 Spring Boot 3.3.5 + JPA + PostgreSQL 完成项目骨架初始化：

| 配置项 | 内容 |
|:-------|:-----|
| 数据源 | PostgreSQL 连接池（HikariCP） |
| JPA 配置 | ddl-auto: update（开发用）、show-sql: true |
| Jackson 配置 | 全局日期格式 `yyyy-MM-dd HH:mm:ss`，JavaTimeModule 注册 |
| 端口 | 8080 |

### 2. 数据持久层编写

| 层 | 文件 | 说明 |
|:---|:-----|:-----|
| Entity | ProjectInfo | 项目信息实体的属性、表映射、通用字段自动维护 |
| Entity | ContractInfo | 合同信息实体，含合同金额 BigDecimal |
| Entity | ContractProjectRef | 合同-项目多对多关联实体 |
| Entity | BoqMain | 工程量清单主表实体，含 DAP 业务时间映射 |
| Entity | BoqDetail | 工程量清单明细实体，含含税单价/数量/层级 |
| Repository | ×5 | 各实体的数据访问接口，含 findBy 业务主键方法 |

所有 Entity 通过 `@PrePersist` / `@PreUpdate` 自动维护 create_time、last_time、f_bizstate、founder、ls_modifier 等通用字段，无需在 Service 层手动赋值。

### 3. 三个同步接口编码

| 接口 | 路径 | 业务逻辑要点 |
|:-----|:-----|:-------------|
| **项目信息同步** | `POST /api/sync/projectInfoList` | 按 project_id 幂等处理，存在则更新所有字段，不存在则新增；支持批量 |
| **合同信息同步** | `POST /api/sync/contractInfoList` | 按 contract_id 幂等，除合同主表外同步维护合同-项目关联关系（ContractProjectRef），更新时先清空旧关联再重建 |
| **工程量清单同步** | `POST /api/sync/boqInfoList` | 按 boq_id 幂等，主表+明细表一对多同时同步，更新时替换全部明细（先删后插），业务时间字段正确映射 `biz_create_time` / `biz_last_update_time` |

### 4. DTO 与响应封装

- 编写 3 个同步请求 DTO（ProjectSyncItem、ContractSyncItem、BoqSyncItem），字段类型与 Entity 一一对应
- 编写统一响应封装 `ApiResult<T>`（含 code、message、data）和逐条反馈 `SyncFeedback`（含 id、status、message）
- 实现 `GlobalExceptionHandler`，统一捕获校验异常并返回标准错误格式

### 5. 编译验证

Maven clean compile 通过，35 个 Java 源文件无编译错误。
